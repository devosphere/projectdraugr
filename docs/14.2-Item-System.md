# 14.2 – Item System

> **Project Draugr**
>
> *An item is a physical object with identity, purpose, and history.*

---

# Purpose

This document defines the Item System used throughout Project Draugr.

While the World Entity System defines **that** a physical object exists, the Item System defines **what** that object is.

Every physical object that can be interacted with is represented by an Item.

Items may be:

- crafted
- harvested
- refined
- traded
- repaired
- modified
- equipped
- stored
- consumed
- destroyed

---

# Relationship with World Entities

Every Item is a World Entity.

However, not every World Entity behaves identically.

The World Entity System manages:

- identity
- persistence
- ownership
- location

The Item System manages:

- functionality
- properties
- usage
- durability
- equipment behavior

---

# Item Identity

Every Item possesses a permanent Item ID.

Example

```text
ITEM-000001
```

The Item ID:

- is globally unique
- never changes
- is never reused
- remains associated with the object until destruction

# Equippable Property

Every Item explicitly defines whether it can be equipped.

```text
Equippable

True
```

or

```text
Equippable

False
```

This prevents invalid situations.

Example

Invalid

```
Equip Horse
```

Horse

```
Equippable

False
```

Simulation Result

Action rejected.

↓

---

# Modification

Items may be modified.

Examples

- sharpen blade
- reinforce handle
- repair clothing
- attach bindings

Modification updates the existing Item.

It does not create a new Item.

---

# Consumption

Consumables disappear after use.

Examples

```
Cooked Fish
```

↓

Eat

↓

Consumed

↓

Entity retired.

---

# Item Destruction

Items may be destroyed through:

- fire
- combat
- weather
- decay
- dismantling
- consumption

Destroyed Items permanently leave the active world.

Their historical record remains.

# Final Principle

An Item is more than an inventory entry.

It is a persistent physical object that occupies space within the world, possesses history, and continues to exist whether or not a Chronicle remembers it.
