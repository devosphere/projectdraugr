# 18.1 - Main UI

> **The interface exists to connect the player with the world, not to replace the world.**

---

# Purpose

This document defines the primary gameplay interface of Project Draugr.

The Main UI is designed around one principle:

> **The world is the focus. The interface only provides awareness and interaction.**

Project Draugr is not designed as a traditional RPG interface.

The player does not control the world through buttons, skill bars, or action menus.

The player communicates intent through the **Action Composer**.

---

# Core UI Philosophy

The player is not controlling a character.

The player is experiencing a Chronicle.

The interface should represent the information a human would realistically know.

The interface provides awareness of:

- Current body condition
- Environmental conditions
- Current time
- Available interaction systems

The interface should avoid exposing unnecessary information.

# Layout Priority

The Main UI follows this priority:

1. Narration
2. Action Composer
3. Body Awareness
4. Secondary Menus

The world always receives the largest visual space.

The interface exists to support the player's understanding of the world.

---

# Environmental Header

The Environmental Header provides external world awareness.

It remains compact and does not use unnecessary labels.

## Example

```text
☀️ 🌱 📅 July 20, 2026 16:34
```

The player interprets the meaning through experience.

The header does **not** display:

- Weather:
- Season:
- Temperature:
- Current Location:

The world should not constantly explain itself.

---

# Header Information

## Weather Icon

Represents current external weather conditions.

### Examples

- ☀️ Clear
- 🌧 Rain
- ⛈ Storm
- ❄ Snow
- 🌫 Fog

---

## Season Icon

Represents the current world season.

### Examples

- 🌱 Spring
- ☀ Summer
- 🍂 Autumn
- ❄ Winter

---

## Date

The in-game calendar follows real-world time.

### Example

```text
July 20, 2026
```

---

## Time

The MVP uses server time as the world clock.

### Example

```text
16:34
```

Future versions may introduce planetary geography and multiple time zones.

---

# Narration Panel

The Narration Panel is the primary world interface.

It describes:

- Environmental conditions
- Events
- Results of player actions
- Discoveries
- Consequences
- Changes in the world

## Example

> You stand at the edge of a forest clearing.
>
> The afternoon wind moves through the trees.
>
> A narrow stream flows nearby.
>
> Animal tracks appear near the muddy shore.

---

# Narration Rules

The narrator does not replace player awareness.

The narrator should not constantly remind players about obvious conditions.

## Incorrect

> Your thirst is increasing.
>
> You should drink water soon.

## Correct

> The Body HUD indicates increasing thirst.

The player decides what to do.

The player has access to the information.

The player is responsible for responding.

---

# Nearby Information

Nearby objects should not function as a permanent minimap.

The world should require observation.

## Default Narration

> A narrow stream flows nearby.

The player may choose:

> I observe my surroundings.

The system may reveal:

```text
Nearby

• Forest Stream
• Fallen Tree
• Wild Plants
• Animal Tracks
• Unknown Footprints
```

Observation is an action.

Knowledge must be earned.

---

# Action Composer

The Action Composer is the primary player interaction system.

## Example

```text
> I inspect the fallen tree for useful materials.
```

The player expresses intent.

The simulation determines:

- Possible actions
- Success
- Failure
- Consequences

---

# Action Philosophy

Project Draugr does not use traditional action buttons.

The player does **not** press:

- Attack
- Gather
- Drink
- Equip
- Harvest

Instead:

The player declares an action.

## Example

> I collect dry branches around the fallen tree.

The simulation evaluates:

- Environment
- Available resources
- Player knowledge
- Tools
- Physical capability
- Current condition

---

# Menu Panel

The Menu Panel contains secondary systems.

## Default State

```text
▸ Chronicle

▸ Equipment

▸ Crafting

▸ Construction

▸ Knowledge

▸ Chronicle Map

▸ Settings
```

Menus remain collapsed until requested.

---

# Menu Interaction

Menus may be opened through:

## Mouse

Click the desired menu option.

## Keyboard Shortcuts

| Key | Menu |
|------|------|
| C | Chronicle |
| E | Equipment |
| K | Knowledge |
| M | Chronicle Map |

Shortcuts only activate when the **Action Composer** is **not** active.

When typing inside the composer:

```text
E
```

is treated as text input.

---

# Information Visibility Principle

Project Draugr follows one rule:

> **Information requires attention.**

The player must intentionally inspect information.

## Examples

### Equipment

Requires opening the Equipment interface.

### Storage

Requires opening the physical container.

### Surroundings

Requires observing the environment.

### Knowledge

Requires learning.

---

# UI Expansion

The Main UI provides the foundation for future interfaces.

Future additions may include:

- Equipment Management
- Storage Management
- Crafting Interface
- Construction Interface
- Knowledge System
- Chronicle Records
- World History

These systems should expand without violating the core philosophy.

---

# Final Rule

The Main UI should always answer:

> **"What does my character realistically know right now?"**

It should never answer:

> **"What information would make the game easier?"**

Project Draugr is a world simulation.

The interface exists to support immersion, not remove uncertainty.
