# 18.5 – Crafting UI

> **Project Draugr**
>
> *Knowledge creates possibility. Hands create reality.*

---

# Purpose

The Crafting UI allows players to create physical objects using materials, tools, knowledge, time, and skill.

Crafting in Project Draugr is **not recipe-driven** in the traditional RPG sense.

The interface does not tell the player everything that can be crafted.

Instead, it represents what the Chronicle currently understands.

---

# Design Philosophy

Project Draugr follows one immutable principle:

> **The player crafts because the Chronicle knows how—not because the game provides a recipe list.**

Crafting is an extension of knowledge.

If the Chronicle has never learned how to weave rope, the option simply does not exist.

The interface never spoils undiscovered crafting possibilities.

---

# Opening the Crafting UI

The Crafting UI only appears when the player intentionally requests it.

Examples:

```text id="c6s4m8"
Open crafting.

Show available crafting.

Craft.
```

The UI does not automatically appear.

---

# Knowledge-Driven Crafting

Available crafting options are determined by the Chronicle's accumulated knowledge.

Different Chronicles may see different crafting options despite possessing identical materials.

Example:

A beginner:

# Materials

The Crafting UI only references materials the Chronicle currently possesses or can physically access.

If a required material is unavailable, it is displayed as missing.

Example:

```text id="srgbof"
Materials

✓ 8 × Plant Fiber

✗ Animal Sinew
```

The UI never spawns missing resources.

---

# Tools

Some crafting requires tools.

Examples:

- Stone Knife
- Hammer
- Needle
- Chisel
- Saw

The UI displays required tools separately from materials.

Possessing materials alone is not sufficient.

---

# Estimated Time

Crafting requires time.

The interface displays an estimated duration.

Example:

```text id="s9p2ba"
Estimated Time

35 Minutes
```

Time advances naturally during crafting.

The world continues evolving while work is performed.

---

# Crafting Failure

Crafting success is not guaranteed.

Results depend upon:

- knowledge
- available tools
- material quality
- environmental conditions
- physical condition
- time

Poor crafting may produce:

- lower quality items
- damaged items
- unusable items
- wasted materials

---

# Item Quality

Crafted items naturally vary in quality.

Examples:

```text id="0ltwxs"
Crude

Poor

Common

Well Crafted

Excellent

Masterwork
```

Quality emerges from the crafting process.

It is never selected directly.

---

# Recipe Discovery

Recipes are never automatically unlocked.

Discovery occurs through:

- experimentation
- observation
- teaching
- books
- previous experience

Two players may discover identical objects through different journeys.

---

# Unknown Possibilities

The Crafting UI intentionally hides unknown possibilities.

Example:

The Chronicle has never discovered pottery.

The UI does not display:

```text id="mnk4m8"
Clay Pot
```

The player cannot be tempted by knowledge they do not possess.

---

# Environmental Requirements

Certain crafting operations require suitable environments.

Examples:

Campfire

Requires:

- fuel
- ignition source
- open space

Pottery

Requires:

- clay
- kiln

Metalworking

Requires:

- furnace
- fuel
- hammer
- anvil

The Crafting UI displays these requirements when relevant.

---

# Physical Output

Crafting always produces physical objects.

Finished items appear in the world.

If the player has no available carrying capacity:

The crafted object remains at the crafting location.

Items never teleport into an invisible inventory.

---

# Legacy

Every crafted object becomes part of the world's history.

Future Chronicles may discover:

- abandoned tools
- handmade weapons
- old containers
- player-built furniture
- engraved equipment

Crafting permanently shapes the world.

---

# Future Expansion

The Crafting UI may later support:

- categories
- search
- favorites
- recently crafted
- blueprint books
- collaborative crafting
- workstation crafting
- production queues

These additions must never violate the core philosophy:

Knowledge determines possibility.

Not the interface.

---

# Final Principle

The Crafting UI is not a recipe encyclopedia.

It is a reflection of what the Chronicle currently understands.

The interface never teaches.

It only reveals knowledge already earned.
