# 14.3 – Storage System

> **Project Draugr**
>
> *Everything must exist somewhere.*

---

# Purpose

This document defines the Storage System used throughout Project Draugr.

The Storage System governs where physical items exist, how they are organized, and how they are retrieved.

Unlike traditional RPG inventories, Project Draugr treats storage as a physical reality.

Items occupy space.

Containers occupy space.

Buildings occupy space.

Nothing exists "inside the player."

---

# Design Philosophy

The Storage System follows one immutable principle:

> **Every physical object must always have a physical location.**

The world always knows where an item exists.

Examples:

- on the ground
- inside a backpack
- inside a storage chest
- hanging on a wall
- placed on a workbench
- inside a warehouse

There is never an "abstract inventory."

---

---

# Storage Types

Storage may exist in different forms.

---

## Personal Storage

Storage carried directly by a Chronicle.

Examples:

- Backpack
- Sling Carrier
- Belt Pouch
- Satchel

Personal storage travels with the Chronicle.

---

## Fixed Storage

Storage permanently placed in the world.

Examples:

- Storage Chest
- Shelf
- Cabinet
- Barrel
- Crate
- Basket

These remain in place until physically moved.

---

## Infrastructure Storage

Large storage integrated into buildings.

Examples:

- Warehouse
- Granary
- Armory
- Library
- Workshop Storage

Infrastructure storage is part of permanent structures.

---

## Temporary Storage

Temporary locations.

Examples:

- Ground
- Campfire Area
- Workbench Surface
- Construction Site

Items remain until moved.

---

# Containers

Containers are Items capable of storing other Items.

Examples:

- Backpack
- Chest
- Basket
- Sack
- Barrel

Containers possess their own:

- weight
- capacity
- condition
- location

Containers are World Entities.

---

# Nested Containers

Containers may exist inside other containers.

Example:

```text
Backpack

↓

Leather Pouch

↓

Bone Needle
```

Or

```text
Warehouse

↓

Storage Chest

↓

Toolbox

↓

Stone Chisel
```

Nested storage allows realistic organization.

---

# Capacity

Every container has finite capacity.

Capacity is determined by:

- volume
- weight
- slot limits (optional)

Items cannot exceed container capacity.

Example:

Small Pouch

Capacity:

```
2 L
```

Attempting to store:

```
Large Log
```

Result:

Storage fails naturally.

---

# Weight

Stored items contribute toward container weight.

Example:

Backpack

```
Empty Weight

1.2 kg
```

Stored:

- Stone Axe
- Rope
- Water Skin

Current Weight:

```
8.4 kg
```

Chronicle carrying weight is calculated from stored contents.

---

# Storage Organization

Players may organize storage however they choose.

Example:

Warehouse

```text
Storage Rack A

Tools
```

Storage Rack B

```text
Food
```

Storage Rack C

```text
Medicine
```

Storage Rack D

```text
Construction Materials
```

The simulation never reorganizes storage automatically.

---

# Item Placement

Items may also exist outside containers.

Examples:

- Hanging on wall
- Leaning against tree
- Placed on workbench
- Displayed on shelf

The exact placement becomes part of the world's persistent state.

---

# Retrieval

To use an item, it must first be retrieved.

Example:

Stone Axe

Location

```text
Warehouse

↓

Storage Rack

↓

Wooden Crate
```

The Chronicle must:

- travel to storage
- locate container
- retrieve item

The simulation does not teleport items.

---

# Searching

Poorly organized storage requires time to search.

Examples

Organized Warehouse

```
Stone Axe

Search Time

5 seconds
```

Disorganized Warehouse

```
Stone Axe

Search Time

5 minutes
```

Organization becomes valuable over long-term survival.

---

# Shared Storage

Settlements may have shared storage.

Ownership may be:

- personal
- communal
- restricted

Examples:

Community Food Storage

Owner:

```
Main Camp
```

Chronicle Personal Chest

Owner:

```
Aldren
```

Future multiplayer rules determine access permissions.

---

# Item Movement

Moving an item always updates:

- Item Location
- Parent Container
- Container Contents

Example

Stone Axe

Before

```text
Backpack
```

↓

After

```text
Workbench
```

The world state immediately reflects the new location.

---

# Storage and Crafting

Crafting stations access nearby storage.

Example:

Workbench

Nearby:

- Timber
- Rope
- Stone Blade

Crafting may consume directly from nearby storage when appropriate.

Future automation may extend this behavior.

---

# Storage Persistence

Storage never resets.

If the Chronicle stores food inside a barrel:

The food remains there until:

- removed
- spoiled
- stolen
- destroyed

Time alone does not delete stored objects.

---

# Relationship with Other Systems

Storage interacts with:

- World Entity System
- Item System
- Equipment System
- Crafting
- Construction
- Infrastructure
- Chronicle System

Storage does not own Items.

It only records where they currently exist.

# Future Expansion

Future versions may introduce:

- locks
- keys
- ownership permissions
- inventory labels

---

# Final Principle

Storage is the physical memory of civilization.

A well-organized settlement saves time.

A forgotten crate may become tomorrow's archaeological discovery.

Nothing disappears simply because the player forgets where it was placed.
