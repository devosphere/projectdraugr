# 18.1 – Main UI

> **Project Draugr**
>
> *The interface exists to connect the player with the world, not to replace the world.*

---

# Purpose

This document defines the primary gameplay interface of Project Draugr.

The Main UI is designed around one principle:

> **The world is the focus. The interface only provides awareness and interaction.**

Project Draugr is not designed as a traditional RPG interface.

The player does not control the world through buttons, skill bars, or action menus.

The player communicates intent through the **Action Composer**.

---

# Core UI Philosophy

The player is not controlling a character.

The player is experiencing a Chronicle.

The interface should represent the information a human would realistically know.

The interface provides awareness of:

- Current body condition
- Environmental conditions
- Current date
- Current time
- Available interaction systems

The interface intentionally avoids revealing hidden information or providing gameplay assistance.

The player must interpret information and make decisions independently.

# Layout Priority

The Main UI follows this information priority:

1. Narration
2. Action Composer
3. Body HUD
4. Menu Panel
5. Environmental Header

The Narration Panel always receives the largest visual space.

The interface exists to support the player's understanding of the world rather than compete with it.

---

# Environmental Header

The Environmental Header provides awareness of the external world.

It remains intentionally compact and avoids descriptive labels.

## Example

```text
☀️ 🌱 📅 July 20, 2026 16:34
```

The player interprets the symbols through experience.

The header intentionally does **not** display labels such as:

- Weather
- Season
- Temperature
- Location

The world should not constantly explain itself.

---

# Header Information

## Weather Icon

Represents the current weather.

Examples:

- ☀️ Clear
- 🌧 Rain
- ⛈ Storm
- ❄ Snow
- 🌫 Fog

---

## Season Icon

Represents the current season.

Examples:

- 🌱 Spring
- ☀ Summer
- 🍂 Autumn
- ❄ Winter

---

## Date

The in-game calendar follows real-world time.

Example

```text
July 20, 2026
```

---

## Time

The MVP uses server time as the world clock.

Example

```text
16:34
```

Future versions may support regional time zones as the simulated world expands.

# Narration Panel

The Narration Panel is the primary interface between the player and the simulated world.

It describes:

- environmental conditions
- visible events
- consequences of player actions
- discoveries
- changes in the surroundings
- reactions from living creatures
- observable world interactions

The narrator never reveals hidden information.

The narrator only describes what the Chronicle can realistically perceive.

---

# Narration Rules

The narrator describes the **external world**.

The narrator does **not** report the Chronicle's internal bodily state.

The following information belongs exclusively to the **Body HUD**:

- Health
- Condition
- Hunger
- Thirst
- Energy
- Temperature
- Wetness
- Bladder
- Bowel
- Hygiene

## Incorrect

> You are becoming thirsty.

> Your bladder is getting full.

> You should probably wash yourself.

## Correct

> The afternoon sun continues beating down through the trees.

> Mud clings to your boots after crossing the stream.

> Smoke from the campfire lingers on your clothing.

The narrator describes the world.

The Body HUD reports the Chronicle's body.

The player is responsible for monitoring it.

---

# Nearby Information

The narration is not a permanent minimap.

Objects, landmarks, and resources become known through observation.

Default narration:

> A narrow stream flows nearby.

The player may choose to perform an observation.

Example:

> I carefully observe my surroundings.

The simulation may then reveal:

```text
Nearby

• Forest Stream
• Fallen Tree
• Wild Plants
• Animal Tracks
• Unknown Footprints
```

Observation is an intentional action.

Knowledge is earned.

---

# Body HUD Responsibility

The Body HUD exists because the player cannot physically inhabit the Chronicle.

Unlike real life, the player cannot naturally perceive:

- hunger
- thirst
- fatigue
- body temperature
- wetness
- bladder pressure
- bowel discomfort
- personal hygiene
- illness
- injury

These sensations are therefore represented by the Body HUD.

The simulation assumes the Chronicle is always aware of these bodily sensations.

The player must monitor them.

Ignoring the Body HUD is considered a player decision.

The narrator never compensates for player negligence.

---

# Action Composer

The Action Composer is the primary method of interacting with the world.

Example:

```text
> I inspect the fallen tree for useful materials.
```

The player expresses intent.

The simulation determines:

- whether the action is possible
- required knowledge
- required tools
- required materials
- required time
- physical consequences
- world consequences

---

# Action Philosophy

Project Draugr does not use traditional action buttons.

There are no:

- Attack buttons
- Gather buttons
- Drink buttons
- Equip buttons
- Harvest buttons

Instead, the player communicates naturally.

Example:

> I collect dry branches around the fallen tree.

The simulation evaluates:

- surrounding environment
- available resources
- Chronicle knowledge
- available tools
- current body condition
- weather
- time
- physical limitations

The world determines the result.

---

# Menu Panel

The Menu Panel provides access to secondary systems.

These systems remain collapsed until intentionally opened.

Default menu:

```text
▸ Chronicle

▸ Equipment

▸ Storage

▸ Crafting

▸ Construction

▸ Knowledge

▸ Chronicle Map

▸ Settings
```

The Menu Panel exists to organize information.

It does not replace player interaction with the world.

---

# Menu Interaction

Menus may be opened using either:

## Mouse

Click the desired menu.

## Keyboard Shortcuts

Example shortcut assignments:

| Key | Menu |
|------|------|
| C | Chronicle |
| E | Equipment |
| I | Storage |
| K | Knowledge |
| M | Chronicle Map |

Keyboard shortcuts are only active while the Action Composer is **not** focused.

If the player is typing:

```text
> Examine the fallen tree.
```

The letter **E** is treated as text rather than a shortcut.

---

# Information Visibility Principle

Project Draugr follows one core rule:

> **Information requires attention.**

The player must intentionally inspect information.

Examples:

Equipment

→ Open Equipment UI

Storage

→ Open the physical container

Knowledge

→ Learn through experience

Maps

→ Open the Chronicle Map

Crafting

→ Open Crafting UI

The interface never forces information onto the player.

---

# UI Expansion

The Main UI provides the foundation for future interface systems.

Future additions include:

- Equipment UI
- Storage UI
- Crafting UI
- Construction UI
- Knowledge UI
- Chronicle UI
- Chronicle Map UI
- Settings UI

Each system expands independently while preserving the overall layout and design philosophy.

---

# Final Principle

The Main UI should always answer:

> **"What does my Chronicle realistically know right now?"**

It should never answer:

> **"What information would make the game easier?"**

Project Draugr is a persistent world simulation.

The interface exists to support awareness.

The world remains uncertain.

The Chronicle remains human.

The player remains responsible.
