# 18.3 - Equipment UI

> **Equipment represents what physically exists on the player's body, not arbitrary RPG equipment slots.**

---

# Purpose

This document defines the Equipment UI system of Project Draugr.

The Equipment UI allows players to view, manage, and understand objects currently worn, attached, or carried on their physical body.

Project Draugr does not use traditional RPG equipment systems.

There are no:

- Helmet slots
- Chest armor slots
- Leg armor slots
- Weapon slots
- Armor tiers

Instead, equipment follows realistic human anatomy.

A person does not equip a *torso item*.

A person wears layers of clothing, carries objects, attaches accessories, and interacts with tools.

---

# Equipment Philosophy

The Equipment system follows one fundamental rule:

> **If an object physically exists on the player's body, the Equipment UI should represent it.**

Equipment exists to answer:

- What am I wearing?
- What am I carrying?
- What is attached to my body?
- What physical protection do I have?
- What items are currently accessible?

The system does **not** exist to display:

- Damage bonuses
- Armor ratings
- RPG statistics
- Character power levels

The simulation determines the consequences.

---

# Equipment Layering

Project Draugr supports realistic equipment layering.

Multiple items may exist on the same body region if they occupy different layers.

## Example

```text
UPPER BODY

Inner Layer
- Linen Shirt

Outer Layer
- Leather Vest

Protective Layer
- Iron Chest Plate
```

The simulation calculates:

- Insulation
- Protection
- Weight
- Mobility Impact
- Environmental Resistance

# Accessories

Accessories exist at their physical locations.

## Neck

Examples:

- Necklace
- Pendant
- Collar

## Shoulders

Examples:

- Amulet
- Shoulder Decoration
- Attached Equipment

## Arms

Examples:

- Bracers
- Arm Guards
- Bracelets

## Fingers

Examples:

- Rings
- Finger Guards
- Small Artifacts

Accessories are **not** automatically powerful items.

A necklace is **not**:

> Defense +5

It is simply:

> Neck → Necklace → Item Entity

The simulation determines its effects.

---

# Finger Equipment Rules

Humans have **10 finger attachment points**.

Each finger can equip **one compatible item**.

**Maximum:** 10 finger equipment positions.

The system prevents:

- Multiple items occupying one finger
- Impossible physical combinations
- Oversized objects

---

# Hands and Tool Interaction

Hands represent active manipulation points.

## Example

```text
Left Hand
- Stone Axe

Right Hand
- Empty
```

The system considers:

- Object Size
- Weight
- Grip Requirements
- Hand Availability

Players cannot:

- Hold unlimited objects
- Equip oversized items
- Use both hands while occupied

---

# Item Entity Compatibility

Every object in Project Draugr exists as an independent item entity.

However, **not every item is equipment.**

Every item defines whether it can be equipped.

## Equippable Example

```json
{
  "item_id": "leather_vest_001",
  "name": "Leather Vest",
  "category": "clothing",
  "equipment": {
    "equippable": true,
    "body_region": "upper_body",
    "layer": "outer"
  }
}
```

---

# Final Rule

The Equipment UI answers:

> **"What physically exists on my body right now?"**

It does **not** answer:

> **"What bonuses does my character have?"**

Project Draugr is not an RPG progression simulator.

Equipment is not power.

**Equipment is existence.**
