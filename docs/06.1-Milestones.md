# 06.1 – Milestones

> **Project Draugr**
>
> *One milestone at a time.*

---

# Purpose

This document defines the measurable milestones required to progress through Project Draugr's development phases.

Unlike the Development Roadmap, this document tracks concrete deliverables rather than long-term vision.

A milestone is considered complete only when every required objective has been satisfied.

---

# Current Phase

## Blueprint Phase

**Status:** Active

Current objective:

Finalize the complete design of Project Draugr before implementation begins.

# Milestone Rules

Every milestone must satisfy the following conditions.

## Complete

No partially implemented systems.

---

## Stable

No milestone should knowingly introduce critical instability.

---

## Persistent

Every completed feature must integrate with the persistent world.

---

## Expandable

Future work should build upon existing systems rather than replace them.

---

## Faithful

Every milestone must remain consistent with the Vision Bible.

---

# Final Principle

Project Draugr is built through disciplined iteration.

Small, complete milestones are preferable to ambitious but unfinished systems.

Every completed milestone should leave the project in a playable and expandable state.

Progress is measured by the quality of the world—not the quantity of implemented features.
